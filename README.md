# 自然语言意图式 SQL 构造器

一个无依赖的本地单页应用。用户通过自然语言意图结构表达“做什么、对谁做、保留什么、排除什么、怎么统计、怎么排序”，系统生成 SQL、参数、人话解释和风险提示。

## 使用方式

直接打开 `index.html` 即可使用。应用不连接真实数据库，也不执行 SQL。

首次使用需要连接已有的本地 JSON 配置文件，或新建一个配置文件。连接完成后，应用会在启动时自动读取该文件，并在数据发生变化时自动保存。

## 当前能力

- 普通模式：查询、统计、业务化表关系、条件树、排序、分页、去重、新增、修改、删除。
- 数据库方言：支持通用 SQL、PostgreSQL、MySQL/MariaDB、SQLite、SQL Server、Oracle、达梦 DM、人大金仓 KingbaseES，自动调整标识符引用、参数占位符、分页语法和建表类型；MySQL 会阻止 FULL JOIN，SQLite 可按 3.39 前/后版本控制 RIGHT/FULL JOIN。
- 大小写策略：可选择标识符“始终引用 / 必要时引用 / 不引用”、标识符“保留 / 统一小写 / 统一大写”，以及文本 `LIKE` 匹配“按数据库规则 / 忽略大小写 / 区分大小写”。
- 高级模式：WITH、计算字段 / CASE / 函数 / 窗口函数、HAVING、UNION。
- 专家模式：建表、改表、索引、视图、事务、权限、Raw SQL。
- 数据字典：支持多个数据字典；可在“管理字典”中批量新增、重命名或删除字典。每个字典下的表和字段支持新增、修改、删除；改名或删除会同步清理当前意图中的失效引用。
- 粘贴导入：在数据字典的“导入”按钮打开弹窗。当前内置 `BIP数据字典格式`：在 YonBIP 中全选字段表格并复制，粘贴后即可自动创建或更新表和字段；弹窗内提供图示教程。
- 本地配置文件：通过浏览器的文件系统访问能力连接或新建 JSON 配置文件。配置文件保存数据字典、当前意图、历史记录及界面设置，并在修改后自动同步；不再提供网页内的导入、导出入口。
- 历史：生成 SQL 后，使用结果区复制按钮左侧的“保存到历史”按钮主动保存。历史弹窗仅用于 SQL 预览，不会跳转或恢复当前构造结果。

## 文件

- `index.html`：主界面结构。
- `styles.css`：三栏工具型布局和响应式样式。
- `assets/bip-data-dictionary-tutorial.png`：BIP 数据字典粘贴导入图示。
- `app-main.js`：意图状态、SQL 生成、参数、解释、风险、历史、本地配置文件同步和 BIP 粘贴导入逻辑。




## 方言与大小写说明

不同数据库的大小写行为不完全由 SQL 语法决定，还会受 collation、兼容模式、MySQL `lower_case_table_names`、SQLite 设置等影响。本工具现在把这些差异拆成可配置策略：

- 标识符引用：影响表名/字段名是否加双引号、反引号或方括号。PostgreSQL、Oracle、达梦、KingbaseES 等在引用标识符后通常要求精确大小写。
- 标识符大小写：可在生成 SQL 时统一输出小写或大写，便于适配未引用建表的库。
- 文本匹配大小写：`包含 / 开头是 / 结尾是` 会按策略生成 `LIKE`、`ILIKE`、`LOWER(...) LIKE LOWER(...)`、MySQL `BINARY ... LIKE` 或 SQLite `GLOB`。
- 风险提示：当当前设置可能受真实数据库环境影响时，会在风险区提示。

## 本轮兼容性修复

- 修复 PostgreSQL `$1` 与 `$10` 等参数预览串号：预览替换会优先处理更长占位符，避免复制 SQL 时条件值被拼错。
- MySQL 不再允许在关系菜单选择 FULL JOIN；切换到 MySQL 时，已有 FULL JOIN 会自动降级为 LEFT JOIN。
- MySQL 在“每页条数=0、跳过>0”时不再生成裸 `OFFSET`，会生成 `LIMIT 18446744073709551615 OFFSET n`。
- 专家模式“改表”按方言生成新增列语法：SQL Server 使用 `ADD column_definition`，Oracle 使用 `ADD (column_definition)`，其余方言保持 `ADD COLUMN`。
- MySQL 事务生成使用 `START TRANSACTION`，不再生成 `BEGIN TRANSACTION`。
- SQLite 增加版本选项：`SQLite < 3.39` 禁用 RIGHT/FULL JOIN；`SQLite 3.39+` 允许。
